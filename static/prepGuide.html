Introduction

Site Prep Overview

Prepping a site for web development is the process of preparing the required assets, information, and documentation for our web development team to be able to successfully implement a design in Aurora. The standards in this guide were developed with input from all constituents to meet the various needs of individuals within Creative Services. Following each step ensures a consistent workflow for your team!

Step 1 | Acquire Files

Aquiring Approved PSD Comps

The first step of the prep process is to acquire the final Photoshop files from the server. Most comp designs go through multiple rounds of client revisions, so it is important to acquire the most recent approved comp.

Client files can be access through the following network directories:

  • Associations:
    • smb://file4-man/Clients/Clients/Associations
  • Chamber, ED, CVB:
    • smb://file4-man/Clients/Clients/Chamber, ED, CVB
  • Cities:
    • smb://file4-man/Clients/Clients/Cities
  • Counties:
    • smb://file4-man/Clients/clients_02/Counties
If you have any concerns contact the Art Director or Project Manager BEFORE you prep the design. It's never fun to have to re-prep a site!

Step 2 | Clean and Organize File

Cleaning and Organizing the PSD Layers

Go through the file and organize the groups and layers if not already. Clean out any unused layers or groups that may have been added during the revision process but are no longer needed.

Step 4 | Create Dropdown and Megamenus

Design Previews for Interactive Navigation Menus

Every CivicPlus site template utilizes both a single-tier dropdown menu and a multiple-tier mega-menu. The design for these elements must be included in the prep assets. Design the styles for these elements and export JPG previews for each interactive state.

Step 5 | Generate Image Assets

Use Photoshop Generator to export image assets for template setup

Images assets include photos, illustrations, icons, and any other visuals that cannot be achieved with HTML/CSS alone. Photoshop will automatically generate assets from layers, but care must be taken to ensure the layers are configured to correctly export the image asset.

Helpful notes:
  • All icons, logos, site IDs, and any element needing transparency should be saved as a PNG format.
  • All photographic images should be saved as JPG format to minimize file size.
  • Banner images should be a minimum of 1800px and ideally 2200px wide.
  • News flash thumnbails should be saved at 150% of comp size by adding 150% before the layer name in Photoshop.
  • Graphic link images should be cropped to the graphic and separated from any shadow or other element that will animate separately
  • Graphics with different shapes/heights should be masked with consistent padding.
  • Site Tools & Share link images should be masked with consistent padding. If text is included it needs to be cut with the icon (on these elements only)..
  • Refrain from cutting text as an image when possible to best comply with ADA regulations.

Photoshop will generate an 'errors.txt' file in the assets folder that is useful for overcoming common issues such as duplicate file names.

Be sure to check your asset files! Even if setup correctly, files can generate incorrectly. Check, check, and check again!
